**Status:** Draft ready for review

## What this is

A one-off (idempotent, re-runnable) seed script that fills the `seo` object fields on 9 Sanity documents with on-brand SEO copy. Runs against the production dataset. Uses `setIfMissing` at field level so **any value a human has already typed in Studio will never be overwritten**.

Script: `scripts/seed-seo.ts`
Pattern reference: `scripts/seed-faqs.ts` (the safe, no-stomp seeder)
Do NOT use: `scripts/seed-sanity.ts` pattern (uses `createOrReplace`, stomps edits)

## Scope

| `landingPage` (fixed `_id`) | ✅ | ✅ | ❌ skipped — needs manual asset upload |
| `bookingPage` (fixed `_id`) | ✅ | ✅ | ❌ |
| `thankYouPage` (fixed `_id`) | ✅ | ✅ | ❌ |

### Out of scope

- **ogImage seeding** — requires uploaded image assets. Upload the brand OG image via Studio, then this script can be extended later, or set manually.
- **Schema changes** — all required SEO fields already exist on the 5 schemas (`landingPage`, `bookingPage`, `thankYouPage`, `reading`, `legalPage`).
- **Next.js metadata wiring** (see "Follow-up" below).

## Safety guarantees

1. **Field-level `setIfMissing`** — if a human edits `seo.metaTitle` in Studio, the next run of the script will SKIP it.
2. **`setIfMissing({ seo: {} })` first** — ensures the parent `seo` object exists before writing dot-path keys. If `seo` already exists with partial data, this is a no-op.
3. **Never `createOrReplace`** — documents are never overwritten; only patched.
4. **Never `set()`** on SEO fields — only `setIfMissing()`.
5. **Missing docs are skipped with a warning**, not errors — script still exits clean for the docs it could patch.
6. **Pre-flight dump** — before any writes, the script logs the current SEO state of every target doc, so you can see exactly what's about to change.

## How to run

```bash
cd www
set -a && source .env.local && set +a && pnpm tsx scripts/seed-seo.ts
```

Requires env vars (already in `.env.local`):
- `NEXT_PUBLIC_SANITY_PROJECT_ID`
- `NEXT_PUBLIC_SANITY_DATASET` (defaults to `production`)
- `SANITY_WRITE_TOKEN` (must have write permission)

## Expected output

Pre-flight section listing every doc's current SEO state (or "document not found"), then a line per field:

```
=== Pre-flight: current SEO state ===
  landingPage: seo.metaTitle=<empty>, seo.metaDescription=<empty>
  reading-soul-blueprint: seo.metaTitle=<empty>, seo.metaDescription=<empty>
  legalPage(privacy): seo.metaTitle="Privacy Policy — Josephine", seo.metaDescription=<empty>

FILL  landingPage.seo.metaTitle

FILL  legalPage(privacy).seo.metaDescription
...

Done. Filled N, skipped M.
```

## Copy source of truth

The SEO copy was generated by a 3-agent brand/SEO council:
- SEO strategist (keyword/intent discipline)
- Brand copywriter (Quiet Archivist voice)
- Spiritual-niche marketing reviewer (persona fit, competitive landscape)

The chosen copy per field is baked into `scripts/seed-seo.ts`. To rewrite, edit the `COPY` constant at the top of that script and re-run. (Remember: a field that a human has since edited in Studio will NOT be overwritten — you'd need to clear it in Studio first.)

Hard constraints applied:
- `metaTitle` ≤ 60 chars
- `metaDescription` ≤ 160 chars
- Banned-word filter per `brand/STYLE_GUIDE.md` §9
